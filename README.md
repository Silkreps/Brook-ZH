# 全球国际工程招标预警系统

生产目标：云端长期运行的内部管理员系统，自动抓取国际工程项目，核验官方来源，筛选中国企业可参与的大型国际公开采购，翻译为中文并通过 Telegram、邮件和站内消息提醒。

## 技术栈
- Next.js：中文后台、仪表盘、三大项目板块、项目详情页和 API。
- Supabase：PostgreSQL、认证、RLS、每日备份和恢复基础。
- Vercel：HTTPS 部署与 Cron。
- n8n：外部调度、失败报警和后续多源编排。
- OpenAI API：官方资料抽取、翻译、分类、风险分析和评分。

## 当前阶段已实现
- 仪表盘首页，覆盖 18 个核心指标：今日新增、符合条件、资格预审、正式招标、前瞻项目、即将截止、延期、补遗、澄清、授标、取消、数据源状态、AI 状态、最近抓取、最近推送、成功率、国家数量和机构数量。
- 三大板块页面：资格预审、正式招标、前瞻项目。
- 项目详情页：中文/英文名称、国家、区域、行业、编号、金额来源、采购阶段、业主、融资机构、工程范围、资格要求、联合体、本地注册、投标保证金、标书费用、截止日期、状态、中国企业参与、AI风险、AI评分和官方链接。
- 自动运行 API：`POST /api/cron/run`。
- Supabase 数据库结构：项目、官方链接、数据源、运行日志、推送、管理员审计、提醒规则、项目事件历史。
- 核心准入规则：国际公开采购、中国企业资格、金额门槛、未截止、未授标、未取消、官方链接和真实性。
- Telegram/邮件推送接口和日报/截止提醒辅助逻辑。
- Vercel Cron 与 n8n 工作流模板。

## 环境变量
复制 `.env.example` 到云平台环境变量。不得将服务端 Key 暴露给前端。

## 下一阶段

## 合并冲突检查
- 使用 `npm run check:conflicts` 或 `node scripts/check-conflicts.mjs` 扫描未解决的 Git 冲突标记。
- 当前环境未提供 `origin` 或本地 `main` 引用，无法在容器内直接执行真实的 `git merge main`；详见 `docs/merge-resolution.md`。
